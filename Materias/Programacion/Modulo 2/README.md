> **"En el Módulo 1 aprendiste a pensar y a mover una sola caja de datos a la vez. Aquí aprendes a organizar cientos de cajas — y a construir tus propias máquinas reutilizables para procesarlas."**
> 
> Bienvenid@ al Módulo 2 de Programación. Cierra dos promesas dejadas desde el inicio del temario: las Funciones (ya vistas conceptualmente en Matemáticas 1.4) por fin llegan a Python real, y las estructuras de datos que necesitas para representar el mundo — listas, diccionarios, tuplas y sets — dejan de ser un misterio.

## 🎯 Objetivos del Módulo

Al terminar este módulo, serás capaz de:

- **Construir bloques de código reutilizables:** declarar funciones propias en Python con `def`, entender parámetros, `return`, y el concepto de scope.
- **Organizar colecciones ordenadas de datos:** dominar listas — su indexado, sus métodos, y cómo recorrerlas eficientemente.
- **Acceder a datos por nombre, no por posición:** usar diccionarios para representar información real de forma clara y con acceso casi instantáneo.
- **Elegir la estructura correcta para cada problema:** entender cuándo usar tuplas (datos inmutables) y sets (sin repetidos, operaciones de conjuntos), en vez de forzar todo a listas.

## 🗺️ Mapa de Ruta (Lo que encontrarás aquí)

Para seguir estructurando el mundo en código, avanzaremos a través de estas lecciones clave:

### 📄 Lección 2.1: Funciones en Python (De la Máquina Matemática al Código Real)

Cerramos la promesa de Matemáticas 1.4. La sintaxis real de `def`, parámetros con valores por default, argumentos con nombre, scope local vs. global, y composición de funciones.

### 📄 Lección 2.2: Listas (Colecciones Ordenadas)

La primera estructura para guardar muchos valores. Indexado, slicing, mutabilidad, los métodos más usados, y una introducción a las list comprehensions prometidas desde el Módulo 1.

### 📄 Lección 2.3: Diccionarios (Pares Clave-Valor)

Acceso por nombre en vez de posición, con una velocidad que viene directo del hashing de Matemáticas 2.3. La base de cómo se representan los datos en JSON y en muchas bases de datos modernas.

### 📄 Lección 2.4: Tuplas y Sets (Cuándo Usar Cada Estructura)

El cierre maestro. Tuplas para datos que nunca deben cambiar, y sets como la implementación directa de la Teoría de Conjuntos de Matemáticas 2.4 — con las mismas operaciones de unión, intersección y diferencia, ahora en código ejecutable.
